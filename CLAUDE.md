# Project Development Guidelines

## Project Overview

This is a capstone project developed during the FlyRank.ai internship using AI-assisted development practices.

## Technology Stack

- Node.js
- JavaScript
- Git
- GitHub
- Cursor AI

## Coding Conventions

- Use clear and descriptive variable names.
- Keep functions small and focused.
- Prefer readable and maintainable code.
- Avoid unnecessary dependencies.
- Add comments only when they improve understanding.

## Git Conventions

Use Conventional Commits.

Examples:

- feat: add a new feature
- fix: resolve a bug
- docs: update documentation
- chore: update project configuration
- refactor: improve code structure

## AI-Assisted Development

AI tools may be used to assist with:

- Code generation
- Documentation
- Debugging
- Code review
- Refactoring

All AI-generated changes should be reviewed and understood before being committed.

## Project Rules

- Never commit secrets or API keys.
- Keep the README updated.
- Keep the project structure organized.
- Test changes before committing them.
